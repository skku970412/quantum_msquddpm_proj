────────────────────────────────
Phase 1. match_corruption 실제 구현
────────────────────────────────

현재 프로젝트에는 `match_corruption` 설정 필드와 forward corruption logging은 있지만, 실제로 random-unitary baseline과 depolarizing forward의 corruption severity를 맞추는 로직은 없다.

이를 구현하라.

목표:
- random-unitary forward와 depolarizing forward를 공정 비교할 수 있게 한다.
- 모델 성능 차이가 단순히 “noise 강도 차이” 때문이 아니라는 점을 보여줄 수 있게 한다.
- 신청서/발표에서 “baseline comparison fairness를 고려했다”고 말할 수 있게 한다.

구현 요구사항:

1. `src/quddpm_lite/noise.py` 또는 적절한 module에 다음 함수 추가

- `measure_target_noisy_fidelity(states, noisy_rhos_or_states, input_mode="density")`
  - target pure states와 noisy density/state 사이의 fidelity를 batch 평균과 표준편차로 계산
  - output:
    - mean_fidelity
    - std_fidelity

- `calibrate_depolarizing_beta_for_target_fidelity(target_fidelity, dim, mode="single_beta")`
  - single_beta mode의 경우 closed-form 사용 가능:
    - F = 1 - beta + beta / dim
    - beta = (1 - F) / (1 - 1/dim)
  - beta는 [0, 1] 범위로 clamp
  - output:
    - calibrated_beta
    - expected_fidelity

- `calibrate_cumulative_beta_end_for_target_fidelity(target_fidelity, noise_steps, beta_start, dim)`
  - cumulative mode에서는 alpha_bar_final이 target fidelity를 맞추도록 beta_end를 근사 탐색
  - linear schedule beta_start → beta_end 가정
  - binary search 사용
  - output:
    - calibrated_beta_end
    - calibrated_alpha_bar_final
    - expected_fidelity

2. `src/quddpm_lite/train.py` 또는 `experiments.py`에 corruption matching pipeline 추가

`match_corruption=True`일 때 다음 절차를 수행하라.

A. 먼저 random-unitary baseline의 final corruption severity를 측정한다.
- train/test subset 중 일부를 사용
- final step `t = noise_steps - 1`
- random-unitary forward process 적용
- target-noisy fidelity mean 측정
- 이 값을 `target_corruption_fidelity`로 둔다.

B. depolarizing model의 beta schedule을 조정한다.
- depolarizing_mode == "single_beta"이면 beta_end 또는 beta_t를 조정
- depolarizing_mode == "cumulative"이면 beta_end를 binary search로 조정
- 조정 결과를 config 또는 run-level local schedule에 반영
- 원래 config를 무조건 overwrite하지 말고, run_config나 metrics에 calibrated 값 기록

C. 각 model row에 다음 컬럼을 저장한다.

- `match_corruption_enabled`
- `corruption_reference_process`
  - 예: "random_unitary"
- `target_corruption_fidelity`
- `actual_forward_fidelity_mean`
- `actual_forward_fidelity_std`
- `corruption_match_error_abs`
- `calibrated_beta_end`
- `calibrated_alpha_bar_final`
- `corruption_match_method`
  - "closed_form_single_beta"
  - "binary_search_cumulative"
  - "none"
- `corruption_match_notes`

3. CNR 처리
- `cnr`은 forward diffusion이 없으므로 corruption matching 대상이 아니다.
- CNR row에는:
  - `forward_process_type="cnr_none"`

  - `corruption_match_notes="CNR is one-step generator comparator; no forward corruption."`
  로 기록하라.

4. README 업데이트

설명해야 할 내용:

Phase 4. ancilla_toy module 추가
────────────────────────────────

공식 QuDDPM 구조에서는 reverse process가 data system + ancilla qubit에 PQC를 적용하고, ancilla measurement/post-selection을 통해 denoising map을 구현하는 방식이다.

현재 프로젝트는 PyTorch density-matrix feature 기반 denoiser 중심이므로, explicit ancilla measurement 구조가 약하다.

이를 보강하기 위해 small-scale `ancilla_toy` model을 추가하라.

목표:
- full QuDDPM 재현이 아니라, official reverse process의 핵심인 ancilla measurement/post-selection을 작은 규모에서 demonstration
- 신청서/발표에서 “공식 구조와의 정합성도 고려했다”고 말할 수 있게 함
- 단, smoke-level로 가볍게 유지

주의:
- Qiskit/PennyLane 의존성을 필수로 추가하지 말라.
- 기존 torch statevector utility로 구현하라.
- 복잡한 density matrix mixed-state 입력까지 완전 지원하려 하지 말라.
- `ancilla_toy`는 pure-state toy demo로 제한해도 된다.
- 제한 사항은 README에 명확히 적어라.

구현 요구사항:

1. 새 파일 추가 가능
- `src/quddpm_lite/ancilla.py`

함수:
- `append_ancilla_zero(states)`
  - input: data statevector `[batch, 2**n]`
  - output: `[batch, 2**(n+1)]`
  - ancilla를 마지막 qubit으로 추가

- `project_ancilla(system_state, data_qubits, outcome=0, eps=1e-12)`
  - system_state shape `[batch, 2**(n+1)]`
  - ancilla qubit은 마지막 qubit으로 가정
  - outcome=0 또는 1에 대해 post-selection
  - output:
    - post_selected_data_state `[batch, 2**n]`
    - success_probability `[batch]`
  - probability가 너무 작으면 eps로 안정화
  - state normalize

- `controlled_or_entangling_layer_with_ancilla(...)`
  - 가능하면 data qubit과 ancilla 사이의 simple entangling operation을 구현
  - 너무 복잡하면 CNOT-like 또는 phase-like toy operation으로 구현

2. `models.py`에 추가
- class `AncillaToyDenoiser(nn.Module)`
- model_name = `"ancilla_toy"`

구조:
- input은 가능하면 noisy statevector
- data state에 ancilla |0> 추가
- trainable rotation layer 적용
- data qubit과 ancilla 사이에 simple entangling layer 적용
- ancilla outcome=0으로 post-select
- output:
  - generated/post-selected data state
  - success_probability

만약 기존 train pipeline이 density matrix를 입력으로 주는 구조라면:
- `ancilla_toy` model 선택 시 config/input_mode를 statevector 또는 pure-state toy로 강제
- 혹은 train branch에서 clean_states와 random-unitary noisy statevectors를 사용
- mixed density input은 지원하지 않아도 되며, unsupported case에서는 명확한 error message를 출력

3. train.py 수정
- model_name == "ancilla_toy" 전용 branch 추가
- loss:
  - `1 - fidelity(clean, generated)`
  - plus success probability penalty
  - 예:
    - `loss = fidelity_loss + lambda_success * mean((1 - success_probability))`
    - lambda_success 기본 0.01 또는 config로 설정
- metrics:
  - `success_probability_mean`
  - `success_probability_std`
  - `post_selection_required=True`
  - `ancilla_qubits=1`

4. generation/evaluation
- ancilla_toy는 우선 reconstruction evaluation만 지원해도 된다.
- generation evaluation까지 가능하면 prior state에 ancilla denoising을 적용해 generated ensemble 평가하라.
- 어렵다면 generation columns는 NaN으로 두고 README에 제한을 명시하라.
