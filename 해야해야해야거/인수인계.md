## 지금까지 보인 경향

현재까지 누적 기준으로는 다음 포인트가 유의미하게 보였음:

- `cnr`는 generation comparator로 매우 강함
- `msquddpm`/`t_msquddpm`는 baseline보다 generation Wasserstein과 total depth 쪽이 더 좋음
- `quddpm_baseline`은 reconstruction fidelity 쪽에서는 shared 계열보다 높게 나오는 경향이 있었음
- `independent_step_quddpm`는 파라미터 수와 runtime이 매우 커서 parameter-efficiency 비교가 아주 잘 드러남

특히 `independent_step_quddpm` 1개 seed 기준:

- `trainable_parameters ≈ 8,012,960`
- `runtime_sec ≈ 697`

shared 계열은 대체로:

- `trainable_parameters ≈ 801k`
- `runtime_sec ≈ 51~52 sec`

즉, parameter-sharing의 비용 절감 효과는 매우 잘 드러남.

### B. 만약 본 실험이 끊겼으면

현재 실행은 detached가 아니므로, 세션이 죽었으면 아마 같이 끊겼을 가능성이 크다.

그 경우 재실행은 detached 권장:

```bash
mkdir -p run_logs
nohup python main.py --preset twohour_readme --results-dir results_readme_twohour > run_logs/twohour_readme.log 2>&1 &
echo $!
```

진행 확인:

```bash
tail -f run_logs/twohour_readme.log
```

README에 꼭 들어가야 할 설명 포인트:

- `cnr`는 generation-only one-step comparator이지 QuDDPM 대체가 아님
- `baseline`은 reconstruction 쪽 강점이 있지만 depth가 큼
- `msquddpm`/`t_msquddpm`는 generation/resource tradeoff가 좋음
- `independent_step_quddpm` 대비 shared 계열이 파라미터를 크게 줄임
- `match_corruption`은 physical equivalence가 아니라 operational fairness calibration임
